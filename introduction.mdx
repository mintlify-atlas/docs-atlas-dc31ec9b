## What is GroceryStore API?

GroceryStore API is a high-performance backend system designed for grocery store inventory and product management. Built with modern .NET technologies, it demonstrates real-world engineering approaches including Vertical Slice Architecture, hybrid data access patterns, and dynamic product metadata validation using PostgreSQL JSONB.

<Warning>
  This project is actively under development. Authentication, Dockerization, and inventory workflows are currently in progress.
</Warning>

## Architecture Overview

### Data Access Strategy

GroceryStore API uses a hybrid approach to data access:

- **Entity Framework Core** - Used for command operations (writes) to leverage change tracking and navigation properties
- **Dapper** - Used for high-performance read queries to minimize overhead and maximize throughput

This separation improves overall performance while maintaining development simplicity for complex write operations.

### Product Metadata System

Products in GroceryStore API support flexible, category-specific attributes:

- **JSONB Storage** - Metadata is stored in PostgreSQL's native JSONB format
- **Recursive Inheritance** - Categories inherit required and optional attributes from parent categories
- **Dynamic Validation** - Attribute constraints (types, ranges, required flags) are retrieved dynamically from the database
- **Result Pattern** - Validation uses a Result pattern to batch errors and avoid exception overhead

### Data Integrity

The system prioritizes predictable data behavior:

- **Soft Deletion** - Products are logically deleted using an `is_active` flag with EF Core Global Query Filters
- **Foreign Key Safety** - All relationships use `Restrict` delete behavior to prevent cascading deletions
- **Data Canonicalization** - Incoming values are automatically normalized to standard formats before storage

### Infrastructure Automation

- **Snake_case Convention** - Database objects automatically converted to PostgreSQL-friendly `snake_case`
- **Compile-Time Registration** - Endpoints and services registered via Source Generators, eliminating reflection overhead
- **Memory Caching** - Category schemas cached in-memory for high-frequency reads

## Use Cases

GroceryStore API is ideal for:

- **Grocery Store Management** - Manage products with category-specific attributes (dairy fat content, alcohol percentage, etc.)
- **Inventory Systems** - Track product batches with FIFO processing and expiration dates
- **E-commerce Backends** - Support flexible product catalogs with dynamic attribute schemas
- **Learning Modern .NET** - Study real-world implementations of Vertical Slice Architecture and hybrid data access
