## Required Software

Before getting started with GroceryStore API, ensure you have the following tools installed on your system.

### .NET 10 SDK

GroceryStore API is built with .NET 10, the latest version of the .NET platform.

<Note>
  .NET 10 is required. Earlier versions (including .NET 8 or 9) are not compatible with this project.
</Note>

**Installation:**

<CodeGroup>
```bash Windows
# Download from Microsoft
winget install Microsoft.DotNet.SDK.10

# Or download installer from:
# https://dotnet.microsoft.com/download/dotnet/10.0
```

```bash macOS
# Using Homebrew
brew install dotnet@10

# Or download installer from:
# https://dotnet.microsoft.com/download/dotnet/10.0
```

```bash Linux
# Ubuntu/Debian
wget https://dot.net/v1/dotnet-install.sh
sudo chmod +x dotnet-install.sh
./dotnet-install.sh --channel 10.0

# Or use package manager:
# https://learn.microsoft.com/dotnet/core/install/linux
```
</CodeGroup>

**Verify Installation:**

```bash
dotnet --version
# Should output: 10.0.x
```

### PostgreSQL

GroceryStore API uses PostgreSQL as its primary database, leveraging JSONB for flexible product metadata.

**Minimum Version:** PostgreSQL 12 or higher (PostgreSQL 14+ recommended)

<Note>
  PostgreSQL 14 or higher is recommended for optimal JSONB performance and query capabilities.
</Note>

**Installation:**

<CodeGroup>
```bash Windows
# Download installer from:
# https://www.postgresql.org/download/windows/

# Or using Chocolatey:
choco install postgresql
```

```bash macOS
# Using Homebrew
brew install postgresql@16
brew services start postgresql@16
```

```bash Linux
# Ubuntu/Debian
sudo apt update
sudo apt install postgresql postgresql-contrib

# Start PostgreSQL service
sudo systemctl start postgresql
sudo systemctl enable postgresql
```
</CodeGroup>

**Verify Installation:**

```bash
psql --version
# Should output: psql (PostgreSQL) 12.x or higher
```

**Post-Installation Setup:**

Create a PostgreSQL user and set password (if needed):

```bash
# Connect to PostgreSQL
sudo -u postgres psql

# Create user (if not exists)
CREATE USER postgres WITH PASSWORD 'your_password';

# Grant privileges
ALTER USER postgres CREATEDB;

# Exit
\q
```

### Entity Framework Core Tools

The EF Core CLI tools are required for running database migrations.

**Installation:**

```bash
dotnet tool install --global dotnet-ef
```

**Verify Installation:**

```bash
dotnet ef --version
# Should output: Entity Framework Core .NET Command-line Tools 10.0.x
```

<Warning>
  If you already have an older version of `dotnet-ef`, update it:
  ```bash
  dotnet tool update --global dotnet-ef
  ```
</Warning>

## System Requirements

### Minimum Requirements

- **OS:** Windows 10/11, macOS 11+, or Linux (Ubuntu 20.04+, Debian 11+, Fedora 33+)
- **RAM:** 4 GB (8 GB recommended)
- **Disk Space:** 2 GB for SDK and tools
- **CPU:** x64 or ARM64 processor

### Recommended Configuration

- **OS:** Latest version of Windows 11, macOS 14+, or Ubuntu 22.04 LTS
- **RAM:** 16 GB or more
- **Disk Space:** SSD with 10+ GB free space
- **CPU:** Multi-core x64 processor (Intel i5/Ryzen 5 or better)

## Verify Your Environment

Run this checklist to ensure everything is properly installed:

<Steps>
  <Step title="Check .NET SDK">
    ```bash
    dotnet --version
    # Expected: 10.0.x
    ```
  </Step>
  
  <Step title="Check PostgreSQL">
    ```bash
    psql --version
    # Expected: psql (PostgreSQL) 12.x or higher
    ```
  </Step>
  
  <Step title="Check EF Core Tools">
    ```bash
    dotnet ef --version
    # Expected: 10.0.x
    ```
  </Step>
  
  <Step title="Test PostgreSQL Connection">
    ```bash
    psql -h localhost -U postgres -d postgres
    # Should connect successfully
    ```
  </Step>
  
  <Step title="Check Git">
    ```bash
    git --version
    # Expected: git version 2.x or higher
    ```
  </Step>
</Steps>
