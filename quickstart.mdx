# Quickstart

Get up and running with the GroceryStore API in just a few minutes. This guide will walk you through setting up the database, running migrations, and making your first API request.

<Note>
  Make sure you've reviewed the [Prerequisites](/prerequisites) before starting.
</Note>

## Setup Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the GroceryStore repository to your local machine:

    ```bash
    git clone https://github.com/9asmodey6/GroceryStore.git
    cd GroceryStore
    ```
  </Step>

  <Step title="Configure the Database">
    Update the connection string in your `appsettings.json` or user secrets with your PostgreSQL credentials:

    ```json appsettings.json
    {
      "ConnectionStrings": {
        "DefaultConnection": "Host=localhost;Database=grocery_db;Username=postgres;Password=YourPassword"
      }
    }
    ```

    <Tip>
      For development, you can use user secrets to keep credentials secure:
      ```bash
      dotnet user-secrets set "ConnectionStrings:DefaultConnection" "Host=localhost;Database=grocery_db;Username=postgres;Password=YourPassword"
      ```
    </Tip>
  </Step>

  <Step title="Apply Migrations">
    Run Entity Framework Core migrations to create the database schema:

    ```bash
    dotnet ef database update
    ```

    This will:
    - Create all necessary tables (`products`, `categories`, `brands`, `countries`, etc.)
    - Apply the snake_case naming convention
    - Seed initial data including categories, attributes, brands, and countries

    <Info>
      The database is automatically pre-seeded with:
      - **Categories**: Dairy, Alcohol, Meat, Beverages, Bakery, and more
      - **Attributes**: Fat Content, Volume, Alcohol %, Weight, Expiration Date, etc.
      - **Brands**: Sample brands for testing
      - **Countries**: Sample countries of origin
    </Info>
  </Step>

  <Step title="Run the Application">
    Start the API server:

    ```bash
    dotnet run --project GroceryStore
    ```

    The API will start on `http://localhost:5000` (or `https://localhost:5001` for HTTPS).
  </Step>

  <Step title="Access API Documentation">
    Open your browser and navigate to:

    ```
    http://localhost:5000/scalar/v1
    ```

    This will open the interactive Scalar API documentation where you can explore and test all endpoints.
  </Step>
</Steps>

## Your First API Request

Now that the API is running, let's make your first request!

### 1. List All Categories

First, retrieve the available categories:

```bash
curl -X GET http://localhost:5000/api/v1/admin/categories
```

**Response:**
```json
[
  {
    "id": 17,
    "name": "Milk",
    "parentId": 8,
    "fullPath": "Dairy > Milk",
    "level": 2
  },
  {
    "id": 8,
    "name": "Dairy",
    "parentId": null,
    "fullPath": "Dairy",
    "level": 1
  }
]
```

### 2. Check Required Attributes

Before creating a product, check which attributes are required for a specific category:

```bash
curl -X GET http://localhost:5000/api/v1/admin/categories/17/metadata
```

**Response:**
```json
[
  {
    "attributeId": 1,
    "name": "Fat Content",
    "dataType": "Decimal",
    "unit": "%",
    "minValue": 0.0,
    "maxValue": 100.0,
    "isRequired": true
  },
  {
    "attributeId": 3,
    "name": "Weight",
    "dataType": "Integer",
    "unit": "g",
    "minValue": 1,
    "maxValue": 10000,
    "isRequired": true
  }
]
```

### 3. Create a Product

Now create your first product with the required attributes:

```bash
curl -X POST http://localhost:5000/api/v1/admin/products \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Whole Milk",
    "description": "Fresh whole milk",
    "price": 3.99,
    "categoryId": 17,
    "brandId": 1,
    "countryId": 1,
    "baseUnit": "l",
    "attributes": [
      { "id": 1, "value": "3.5" },
      { "id": 3, "value": "1000" }
    ]
  }'
```

**Response:**
```json
{
  "id": 1,
  "name": "Whole Milk",
  "description": "Fresh whole milk",
  "price": 3.99,
  "categoryId": 17,
  "brandId": 1,
  "countryId": 1,
  "sku": "MIL-847293",
  "baseUnit": "l",
  "isActive": true,
  "createdAt": "2024-03-05T10:30:00Z",
  "updatedAt": "2024-03-05T10:30:00Z",
  "metadata": {
    "1": "3.5",
    "3": "1000"
  }
}
```

<Note>
  The **SKU is auto-generated** based on the category name prefix and a random 6-digit number. In this example, "MIL" comes from "Milk".
</Note>

## Troubleshooting

<Accordion title="Database connection failed">
  - Verify PostgreSQL is running: `pg_isready`
  - Check your connection string in `appsettings.json` or user secrets
  - Ensure the database user has CREATE DATABASE permissions
</Accordion>

<Accordion title="Migration errors">
  - Make sure EF Core tools are installed: `dotnet tool list -g`
  - Try cleaning and rebuilding: `dotnet clean && dotnet build`
  - Check if the database already exists and drop it if needed
</Accordion>

<Accordion title="Port already in use">
  - Change the port in `Properties/launchSettings.json`
  - Or kill the process using the port: `lsof -ti:5000 | xargs kill` (macOS/Linux)
</Accordion>
